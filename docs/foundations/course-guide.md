# 授業ガイド

この章では、IW3IP の教材を授業や演習でどのように扱うかを整理します。

個々のサンプルを順番に動かすだけでも学べますが、授業として扱う場合は、どこで全体像を示し、どこで手を動かし、どこで振り返るかを最初に決めておくと進めやすくなります。

## 想定対象

- 情報系学部の学部2-4年生
- 「ネットワーク」「Web/API」「データベース」「プログラミング」の基礎を履修中または履修済み

## この教材で身につく力

- 分散データ共有の基本構成を説明する力
- 同意（Consent VC）にもとづくアクセス判定の理解
- オンチェーン/オフチェーン分離の設計意図の理解
- 実験ログを根拠に検証結果を報告する力

## 学習の進め方

基本的な学習とハンズオンは、このサイト内の流れを順に追うことで進められるように構成しています。

- このサイトで主に扱う内容:
  - 基本用語の理解
  - システム全体像の理解
  - 最短起動
  - Hands-on 実行
  - 結果の読み取り
- 外部資料を読むと役立つ場面:
  - 仕組みをさらに深掘りしたいとき
  - レポートや卒業研究に向けて背景調査を行うとき
  - 標準仕様や原典を確認したいとき

授業では、最初から外部資料に広げすぎるより、まずサイト内の説明とサンプルで共通の前提をそろえる方が理解しやすくなります。

## 15コマ授業での位置づけ（例）

| 回 | テーマ | 本サイトの対応章 |
|---|---|---|
| 1-2 | 導入・背景 | Platform Overview |
| 3-4 | ブロックチェーン基礎 | Blockchain Basics / Hardhat Basics |
| 5-6 | SSI/DID/VC基礎 | SSI/DID/VC Basics |
| 7-9 | 実装演習 | Workshop + Hands-on |
| 10-12 | 検証・比較 | Operations + FAQ |
| 13-15 | 最終発表 | Roadmap + References |

## 推奨評価方法

- 実行再現（40%）: 手順通り動作させられるか
- 理解説明（40%）: なぜ `allow/deny` になるか説明できるか
- 改善提案（20%）: Phase 2/3に向けた提案の妥当性

## レポート最小要件

- 実行環境（OS、Dockerバージョン）
- 実行コマンドと出力ログ
- `allow` ケースと `deny` ケースの比較
- 改善提案（最低1件）
