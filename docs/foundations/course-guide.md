# 授業ガイド

この章では、IW3IP の教材を授業や演習でどのように扱うかを整理します。

個々のサンプルを順に動かすだけでも学べますが、授業では「どこで全体像を示し、どこで手を動かし、どこで振り返るか」を先に決めておく方がスムーズです。

## 想定対象

- 情報系学部の学部2-4年生
- 「ネットワーク」「Web/API」「データベース」「プログラミング」の基礎を履修中または履修済み

## この教材で身につくこと

- 分散データ共有の基本構成を説明できる
- Consent VC にもとづくアクセス判定を理解できる
- オンチェーン/オフチェーン分離の設計意図を理解できる
- 実験ログを根拠に検証結果を報告できる

## 学習の進め方

基本的な学習とハンズオンは、サイト内の流れに沿って進められます。

- サイト内で扱う内容: 基本用語、システム全体像、最短起動、Hands-on 実行、結果の読み取り
- 外部資料が役立つ場面: 仕組みの深掘り、レポートや卒論の背景調査、標準仕様の原典確認

授業では、最初から外部資料に手を広げるよりサイト内の説明で前提をそろえてから進める方が効率的です。

## 15コマ授業での位置づけ（例）

| 回 | テーマ | 本サイトの対応章 |
|---|---|---|
| 1-2 | 導入・背景 | Platform Overview |
| 3-4 | ブロックチェーン基礎 | Blockchain Basics / Hardhat Basics |
| 5-6 | SSI/DID/VC基礎 | SSI/DID/VC Basics |
| 7-9 | 実装演習 | Workshop + Hands-on |
| 10-12 | 検証・比較 | Operations + FAQ |
| 13-15 | 最終発表 | Roadmap + References |

## 推奨評価方法

- 実行再現（40%）: 手順通り動作させられるか
- 理解説明（40%）: なぜ `allow/deny` になるか説明できるか
- 改善提案（20%）: Phase 2/3に向けた提案の妥当性

## レポート最小要件

- 実行環境（OS、Dockerバージョン）
- 実行コマンドと出力ログ
- `allow` ケースと `deny` ケースの比較
- 改善提案（最低1件）
