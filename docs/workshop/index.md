# Workshop（進行設計）

この章は、講師・TAがワークショップ全体を進行するための資料です。

## Workshop と Hands-on の関係

- Workshop: 全体設計（何を、いつ、どの順番で進めるか）
- Hands-on: 参加者が実際に手を動かす実習手順

運用上は次のように使います。

1. Workshopで全体説明と分岐条件を示す
2. Hands-onで参加者が各自のデバイス条件に合わせて実行する
3. Workshopに戻って成果共有と発展課題を行う

## 推奨タイムライン（120分）

1. 0-10分: イントロ（システム構成の説明）
2. 10-30分: 環境確認
3. 30-55分: 最短起動
4. 55-95分: Hands-on（HUSKYLENS2 または Webcam）
5. 95-110分: スマホ閲覧と購入確認
6. 110-120分: まとめと発展課題
